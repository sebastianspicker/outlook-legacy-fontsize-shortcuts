# Contributing

Thanks for considering a contribution! Please follow these guidelines to keep changes reviewable and safe.

## Quickstart

```bash
make tools
make check
```

## Pull Requests

- Keep changes small and focused.
- Add or update tests for behavioral changes.
- Update documentation when behavior or configuration changes.
- Ensure `make check` passes locally.

## Issues

- Use the bug/feature templates when opening issues.
- Include environment details and reproduction steps.
